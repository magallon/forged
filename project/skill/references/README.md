# Referencias del Proyecto

Este directorio contiene documentación detallada del proyecto que las IAs consultan bajo demanda. No se carga completa en cada sesión — la skill dirige a la IA al archivo relevante según la tarea.

---

## Archivos recomendados

| Archivo | Contenido | Cuándo lo consulta la IA |
|:--------|:----------|:-------------------------|
| `codebase-map.md` | Inventario de componentes, funciones, hooks y utilidades existentes | Antes de crear algo nuevo |
| `handbook.md` | Visión general, arquitectura, historia y decisiones del proyecto | Cuando necesita contexto amplio del proyecto |
| `business.md` | Modelo de negocio, pricing, políticas, competencia | Cuando implementa features relacionadas con planes, billing o límites |
| `tech-stack.md` | Stack técnico, infraestructura, pipeline, decisiones de tecnología | Cuando necesita saber qué tecnologías usa el proyecto y por qué |

No es obligatorio tener todos. Crea solo los que tu proyecto necesite. Puedes agregar otros según surjan: `api-contracts.md`, `design-system.md`, `security-policies.md`, etc.
